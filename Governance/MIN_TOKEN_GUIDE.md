# Guía de Tokenización Mínima

## Objetivos
- Reducir el consumo de tokens manteniendo la cobertura de requisitos esenciales.
- Estandarizar la extracción de datos clave para consultas rápidas y coherentes.
- Garantizar que la compresión priorice claridad y trazabilidad sobre la longitud absoluta.

## Prioridades de información
1. **Crítico obligatorio**: decisiones de gobernanza, políticas vigentes, acuerdos firmados.
2. **Contexto operativo**: entregables activos, dependencias, métricas de avance.
3. **Referencia opcional**: historiales, anexos extensos y discusiones informales; resumir y enlazar.

## Convenciones de formato
- Encabezados jerárquicos en Markdown (`#`, `##`, `###`).
- Listas numeradas para pasos o prioridades; listas con viñetas para ejemplos.
- Resúmenes de una sola frase al inicio de cada sección.
- Campos estructurados en bloques tipo tabla cuando haya valores repetitivos.

## Inventario de datos por categoría
| Categoría    | Datos que deben permanecer | Datos vinculables externamente |
|--------------|----------------------------|--------------------------------|
| Gobernanza   | Estatutos, roles vigentes, registros de votación y quórum, obligaciones legales. | Documentos de referencia legal, minutas completas, grabaciones de asambleas. |
| Proyecto     | Alcance aprobado, roadmap vigente, KPIs comprometidos, presupuesto asignado. | Informes detallados, prototipos interactivos, documentación técnica extensa. |
| Tarea        | Objetivo, responsable, estado, próximos hitos, requisitos bloqueantes. | Evidencia multimedia, discusiones largas, enlaces a tickets externos. |

## Validación con el equipo
- Presentar la guía en la próxima reunión de gobernanza para revisión colectiva.
- Recoger confirmaciones de cada rol responsable (producto, técnico, legal) sobre la suficiencia de la información.
- Documentar ajustes acordados y actualizar la guía antes de aplicarla de forma general.
